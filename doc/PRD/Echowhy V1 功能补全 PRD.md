   ## P0 核心功能补全（必须优先实现）
   ### 1. 冷启动梯子（Cold-Start Ladder）过渡态
   **功能描述**：用户成功导入知识源后，先进入项目概览过渡页，提供入门引导，不直接跳转到具体主题。

   **交互逻辑**：
   1.  用户点击"Attach a source"并完成导入后，自动跳转到本页面
   2.  页面结构（完全使用现有视觉风格）：
       - 顶部：显示导入的项目名称（如"RBAC Learning Project"）
       - 中间上半部分：AI生成的项目概述（1-2段文字）
       - 中间下半部分：3个可点击的引导问题卡片（半透毛玻璃样式，与现有源码面板一致）
       - 底部：自定义问题输入框，提示"或者，你也可以问任何你想了解的问题..."
       - 右侧：项目文件结构树（只显示一级目录，如`server/`、`web/`、`docs/`）
   3.  用户操作：
       - 点击任意引导问题卡片 → 进入对应的学习主题页面
       - 输入自定义问题 → 进入自定义主题
       - 点击右侧文件结构中的任意文件 → 生成以该文件为核心的学习主题

   **技术要求**：
   - 使用现有mock数据中的RBAC项目信息作为默认演示内容
   - 所有元素样式严格对齐现有设计
   - 页面跳转使用平滑过渡动画

   **验收标准**：
   - 导入源码后自动跳转
   - 3个引导问题点击后能正确进入学习主题
   - 自定义问题和文件点击功能正常

   ### 2. 完整AI反馈系统（即时悬浮+历史内联，解决空间冲突）
   **功能描述**：区分"即时判答"和"历史沉淀"两种状态，既保持阅读流纯净，又解决历史问题展开后无内容的逻辑冲突。

   **交互逻辑**：
   1.  **即时判答阶段（右侧悬浮）**：
       - 用户在中间内容区输入答案并点击`[ CHECK ]`
       - 右侧顶部自动弹出AI反馈卡片（半透毛玻璃样式，与现有源码面板一致）
       - 反馈结构统一：
         ```
         [分数/等级]
         👍 What landed well:
         - 正确点1
         - 正确点2

         ⚠️ What feels unclear:
         - 模糊点1
         - 模糊点2

         ❌ What's still missing:
         - 缺失点1
         - 缺失点2

         💡 A good next step:
         下一步建议
         ```
       - 三种状态标识：
         - **正确（score ≥ 80）**：绿色等级标识，显示"Good!"或"Strong!"
         - **部分正确（60 ≤ score < 80）**：黄色等级标识，显示"Partial"
         - **错误（score < 60）**：红色等级标识，显示"Weak"
       - 卡片右上角有`×`关闭按钮

   2.  **历史沉淀阶段（内联极简）**：
       - 用户关闭右侧悬浮卡片后，系统自动提取AI反馈中的核心总结（`💡 A good next step`部分），精简成一句话
       - 将这句话固化在中间问题框的输入框下方，使用小号灰色文字显示
       - 问题框自动折叠，只显示问题标题和核心总结

   3.  **阻塞规则（解决悲观路径漏洞）**：
       - **Score ≥ 60（通过）**：关闭卡片后，问题框折叠，左侧对应节点从`pulsing`变为`lit`状态，平滑展开下一段内容
       - **Score < 60（未通过）**：关闭卡片后，问题框**不折叠**，节点**保持pulsing状态**，下一段**不展开**
       - 在输入框下方显示提示：`This doesn't seem quite right. Want to try again or reveal the answer?`
       - 提供两个按钮：`[ Try again ]` 和 `[ Reveal answer ]`
       - 点击`Reveal answer`：自动填充正确答案到输入框，用户确认后可继续

   **技术要求**：
   - 悬浮卡片固定在右侧面板顶部，z-index高于源码面板
   - 卡片宽度与右侧源码面板一致
   - 不同状态使用对应的主题色（绿/黄/红）
   - 内联历史反馈使用小号灰色文字，不抢正文注意力

   **验收标准**：
   - 点击CHECK后，右侧正确弹出反馈卡片
   - 三种状态显示正确
   - 关闭卡片后，中间内容区正确显示极简历史反馈
   - 分数<60时，正确阻塞下一段内容
   - 节点状态正确同步

   ### 3. 学习角度选择器（左侧顶部下拉式，优化空间利用）
   **功能描述**：在左侧星座链侧边栏顶部添加下拉菜单，提供多个学习方向选择，节省垂直空间，把最大画布留给星座链。

   **交互逻辑**：
   1.  位置：左侧星座链区域的最顶部
   2.  样式：极简文字下拉菜单，显示为 `Current Angle: [ Request flow ▼ ]`
   3.  点击下拉箭头，展开4个选项：
       - `Request flow`（请求流程）
       - `Responsibility split`（职责划分）
       - `JWT timing`（JWT时机）
       - `My own why`（自定义问题）
   4.  默认选中第一个角度
   5.  用户点击不同选项，中间内容区平滑切换到对应的问题链
   6.  点击`My own why`，在中间内容区底部显示自定义问题输入框

   **技术要求**：
   - 下拉菜单样式与现有按钮风格一致
   - 展开的选项列表使用半透毛玻璃样式
   - 切换角度时，使用平滑的内容替换动画
   - 下拉菜单固定在左侧顶部，不随页面滚动

   **验收标准**：
   - 左侧顶部正确显示下拉菜单
   - 点击能正确展开和收起选项列表
   - 选择不同角度能正确切换对应的问题链
   - 自定义问题功能正常

   ### 4. 锚点双向联动（懒加载完整文件+沙盒约束，解决布局崩塌）
   **功能描述**：默认显示轻量代码片段，用户主动点击才加载完整文件，严格限制空间，防止撑爆右侧工作台。

   **交互逻辑**：
   1.  **默认状态（轻量片段）**：
       - 右侧源码面板只显示相关的代码片段（与现有完全一致）
       - 片段顶部显示完整文件路径
       - 片段底部新增小号灰色按钮：`📂 View full file`（hover时变为蓝色）
   2.  **完整文件模式（沙盒约束）**：
       - 点击`📂 View full file`按钮 → 面板内容替换为完整源文件，带行号，支持独立滚动
       - **严格空间约束**：完整文件依然被限制在`max-h-96`（约384px）的高度内，用户在局部滚动条内翻阅
       - 按钮变为`📄 Show snippet`，点击可切回片段模式
       - 加载时显示简单状态：`Loading file...`
   3.  **正向联动（中间→右侧）**：
       - 鼠标悬浮中间代码词 → 右侧临时显示对应片段
       - 点击代码词 → 将该片段固定到右侧工作台
       - 若已在完整文件模式 → 自动滚动到对应行并高亮（淡蓝色背景，持续2秒）
   4.  **反向联动（右侧→中间）**：
       - 点击片段/完整文件中的任意一行 → 中间内容区自动平滑滚动到对应解释段落
       - 该段落高亮显示（淡蓝色背景，持续2秒）

   **技术要求**：
   - 完整文件仅在用户点击按钮后加载
   - 两种模式的切换使用淡入淡出动画
   - 行号与实际代码文件完全一致
   - 严格遵守`max-h-96`高度限制，不允许突破

   **验收标准**：
   - 默认显示片段，点击按钮能正确切换到完整文件
   - 完整文件高度不超过384px，支持内部滚动
   - 行号正确
   - 正向和反向联动功能正常
   - 两种模式下的高亮效果正确

   ## P1 重要体验补全
   ### 1. 多源工作台溢出优化
   **功能描述**：解决固定过多代码块导致右侧面板过长的问题。

   **交互逻辑**：
   1.  在右侧`SOURCE WORKBENCH`标题右侧新增小号灰色文字按钮：`[ Clear All ]`（hover变蓝）
   2.  每个源码卡片内部添加独立滚动条，限制最大高度为`max-h-96`（约384px）
   3.  卡片内容超过最大高度时，只显示前10行，其余通过滚动查看
   4.  点击`Clear All` → 清空右侧所有固定的源码卡片

   **技术要求**：
   - 按钮样式与现有关闭按钮一致
   - 滚动条使用自定义样式，对齐现有设计

   **验收标准**：
   - 长代码卡片能正常内部滚动
   - Clear All按钮能正确清空所有源码
   - 单个卡片的关闭按钮功能正常

   ---

   ### 2. 学习终点成就感闭环
   **功能描述**：用户完成主题所有问题后，提供明确的完成反馈和下一步指引。

   **交互逻辑**：
   1.  用户答完最后一个问题并关闭AI反馈后：
       - 左侧星座链所有节点变为`lit`状态，所有连线变为亮蓝色实线（保留现有折线样式）
       - 内容区最底部出现总结卡片（半透毛玻璃样式）
   2.  总结卡片内容：
       ```
       🎉 Topic Mastered!
       You have completed the "Request flow" angle of this topic.

       What would you like to do next?
       [ Explore another angle ]  [ Return to Library ]  [ Ask a follow-up question ]
       ```
   3.  按钮功能：
       - `Explore another angle` → 自动切换到左侧第一个未完成的角度
       - `Return to Library` → 跳转到学习库页面
       - `Ask a follow-up question` → 显示自定义问题输入框

   **技术要求**：
   - 总结卡片样式与现有源码面板一致
   - 节点全部点亮时添加短暂的淡入动画

   **验收标准**：
   - 完成最后一个问题后，所有节点正确点亮
   - 总结卡片正确显示
   - 三个按钮功能正常

   ---

   ### 3. 一键返回当前焦点
   **功能描述**：用户滚动远离当前问题时，提供快速返回入口。

   **交互逻辑**：
   1.  当用户视口完全离开当前`pulsing`状态的问题区域时，页面右下角出现悬浮按钮：`↓ Return to current question`
   2.  点击按钮 → 页面平滑滚动到当前问题位置
   3.  视口回到当前问题区域时，按钮自动消失

   **技术要求**：
   - 按钮使用半透毛玻璃样式，圆角设计
   - 滚动检测使用Intersection Observer API
   - 滚动动画时长300ms

   **验收标准**：
   - 滚动离开时按钮正确出现
   - 点击能正确滚动到当前问题
   - 滚动回来时按钮自动消失

   ---

   ### 4. 已回答问题折叠功能
   **功能描述**：允许用户折叠已回答的历史问题，保持页面整洁。

   **交互逻辑**：
   1.  所有已回答的问题（状态为`lit`），在标题右上角新增小号灰色`-`按钮（hover变蓝）
   2.  点击`-` → 问题框折叠，只显示问题标题和极简历史反馈，按钮变为`+`
   3.  点击`+` → 问题框展开，显示完整的AI反馈内容，按钮变为`-`
   4.  新回答的问题默认自动折叠

   **技术要求**：
   - 按钮样式与现有关闭按钮一致
   - 折叠/展开使用平滑高度过渡动画

   **验收标准**：
   - 已回答问题右上角显示折叠按钮
   - 点击能正确折叠/展开
   - 折叠后正确显示极简历史反馈
   - 新回答的问题默认折叠

   ---

   ### 5. 星座链完整交互
   **功能描述**：让左侧星座链成为真正的导航工具，**保留现有折线样式，不使用任何贝塞尔曲线**。

   **交互逻辑**：
   1.  节点状态：
       - `dim`：未回答/跳过的问题，灰色空心圆点
       - `pulsing`：当前正在回答的问题，带呼吸动画的蓝色实心圆点
       - `lit`：已回答的问题，亮蓝色实心圆点
   2.  连线状态：
       - 已完成路径：亮蓝色实线（保留现有折线）
       - 未完成路径：灰色虚线（保留现有折线）
   3.  点击任意非当前节点 → 中间内容区平滑滚动到对应的问题位置

   **技术要求**：
   - 所有连线均为直线/折线，禁止使用曲线
   - 呼吸动画周期2秒，与现有一致
   - 点击跳转使用300ms平滑滚动

   **验收标准**：
   - 三种节点状态正确显示
   - 连线均为折线，无曲线
   - 点击非当前节点能正确跳转

   1.  **源码文件名完整显示**：将截断的文件名改为换行显示

   ## 通用要求
   1.  **严格保留现有UI**：所有已实现的视觉元素、布局、颜色、字体均不做任何修改
   2.  **新增元素对齐现有风格**：所有新增按钮、卡片、标签均使用与现有完全一致的样式
   3.  **动画统一**：所有过渡动画时长控制在200-300ms之间
   4.  **明暗模式支持**：所有新增元素均需同时支持浅色和深色模式
   5.  **功能隔离**：新增功能与现有功能完全隔离，不影响任何已实现的交互
