# Echowhy V1.5 三种可插入卡片差异化设计 PRD（双主题适配）

## 版本：1.0

## 核心设计原则

**不破坏现有极简风格、不新增任何装饰元素、严格使用产品现有色板、不改变卡片整体布局**。仅通过「材质底色 + 左侧色条 + 排版密度」三个维度建立自然的视觉层级，解决 "三种卡片无法快速区分、阅读疲劳" 的问题。

------

## 一、核心定位与差异化依据

三种卡片本质完全不同，差异化设计严格围绕其信息属性展开，不做任何无意义的装饰：

| 卡片类型               | 信息属性 | 核心定位           | 视觉层级           |
| :--------------------- | :------- | :----------------- | :----------------- |
| My Question 用户问答卡 | 主观交互 | 用户思考的核心区域 | 最高（浮在最上层） |
| Feedback 评估反馈卡    | 系统指导 | 修正思考的批注     | 中等（引导层）     |
| Source 源码插入卡      | 客观引用 | 辅助阅读的参考材料 | 最低（背景层）     |

------

## 二、双主题视觉规范

### 1. 黑夜主题

| 卡片类型      | 左侧色条                         | 背景材质         | 背景透明度                         |
| :------------ | :------------------------------- | :--------------- | :--------------------------------- |
| My Question   | `border-l-2 border-blue-500/70`  | 纯黑毛玻璃       | `bg-black/50 backdrop-blur-sm`     |
| Feedback 评估 | `border-l-2 border-amber-500/70` | 极淡琥珀色毛玻璃 | `bg-amber-950/20 backdrop-blur-sm` |
| Source 源码   | `border-l-2 border-cyan-500/70`  | 极淡青色毛玻璃   | `bg-cyan-950/15 backdrop-blur-sm`  |

### 2. 白天主题

| 卡片类型      | 左侧色条                         | 背景材质   | 背景透明度       |
| :------------ | :------------------------------- | :--------- | :--------------- |
| My Question   | `border-l-2 border-blue-600/70`  | 纯白       | `bg-white`       |
| Feedback 评估 | `border-l-2 border-amber-600/70` | 极淡琥珀色 | `bg-amber-50/30` |
| Source 源码   | `border-l-2 border-cyan-600/70`  | 极淡灰蓝色 | `bg-slate-50/50` |

**设计说明**：

- 所有颜色均来自产品现有色板，无任何新增杂色
- 背景透明度梯度严格对应视觉层级：用户问题最实 → AI 反馈次之 → 源码最透
- 左侧色条仅 2px 宽，极其细微但辨识度极强，不破坏整体简洁感

------

## 三、内部排版差异化规范（不改变布局，仅优化细节）

### 1. My Question 用户问答卡（保持核心高亮）

- 完全保留现有所有样式、布局、字号
- 输入框边框颜色与左侧蓝色色条统一
- `[ CHECK ]` 按钮保持现有样式不变
- 左上角标签：`MY QUESTION`（蓝色大写小字）

### 2. Feedback 评估反馈卡（强化结构化）

- 头部得分区域：`Weak · 18` 文字加粗，字号不变
- 小标题（What landed well / What feels unclear）：字重从 400 提升至 500，字号不变
- 列表项：保持现有文字样式，不新增任何图标
- 左上角标签：`FEEDBACK INSERT`（琥珀色大写小字）
- 禁止将分数提取到右上角，禁止新增任何徽章或装饰

### 3. Source 源码插入卡（降级为参考材料）

- 代码区域字号：整体缩小 1px，保持等宽字体不变
- 文件路径头部：字号缩小 1px，颜色稍微变浅
- 代码高亮保持现有样式不变
- 左上角标签：`SOURCE INSERT`（青色大写小字）
- 禁止将代码区域边框改为虚线，禁止新增任何代码装饰

------

## 四、拖动与插入交互规范

### 1. 拖动时视觉反馈

- 拖动过程中，卡片整体缩小 15%，保持左侧色条可见

- 插入提示线颜色与该卡片的左侧色条完全一致

  - 拖用户卡片：蓝色提示线
  - 拖反馈卡片：琥珀色提示线
  - 拖源码卡片：青色提示线

  

- 吸附到可插入位置时，提示线变粗 1px，亮度提高 30%

### 2. 插入后状态

- 所有卡片插入后，自动与现有内容保持统一的上下间距
- 卡片的圆角、边框粗细、阴影样式完全统一，仅背景和左侧色条不同

------

## 五、折叠状态规范

折叠后依然保持各自的视觉特征，无需展开即可快速识别：

| 卡片类型      | 折叠后显示内容               | 视觉特征           |
| :------------ | :--------------------------- | :----------------- |
| My Question   | `[❓] 问题标题`               | 左侧保留蓝色色条   |
| Feedback 评估 | `[💡] Weak · 18/100`          | 左侧保留琥珀色色条 |
| Source 源码   | `[📄] AuthService.java:18-22` | 左侧保留青色色条   |

------

## 六、绝对禁止项（严格遵守）

1. 禁止给不同卡片使用不同的圆角、边框粗细、阴影样式
2. 禁止新增任何专属图标（{ }、⚡、? 等）
3. 禁止使用产品现有色板以外的任何颜色
4. 禁止改变任何卡片的整体布局和结构
5. 禁止将代码区域边框改为虚线
6. 禁止将反馈卡片的分数提取到右上角做徽章
7. 禁止调整卡片之间的默认上下间距

------

## 七、验收标准

### 通用验收

1. 扫一眼左侧色条，即可在 0.5 秒内区分三种卡片类型
2. 视觉层级自然形成，用户注意力自动聚焦在自己的问题上
3. 所有卡片的圆角、边框、阴影完全统一
4. 拖动时提示线颜色与卡片类型对应
5. 折叠后依然可以快速识别卡片类型

### 黑夜主题验收

1. 用户卡片背景为半透黑色，反馈为极淡琥珀色，源码为极淡青色
2. 左侧色条颜色符合规范，清晰可见但不突兀
3. 代码区域字号比正文小 1px

### 白天主题验收

1. 用户卡片背景为纯白，反馈为极淡琥珀色，源码为极淡灰蓝色
2. 左侧色条颜色符合规范，清晰可见但不突兀
3. 反馈卡片小标题字重为 500，得分文字加粗
