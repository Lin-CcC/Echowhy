# Echowhy V1.5 Library 学习库终极抛光 PRD

## 版本：1.0 (Dark Space Edition)

## 核心原则 (The Golden Rules)

1. **坚守深空全息美学**：全局采用毛玻璃（Glassmorphism）与青色阶（Cyan），禁止使用 2px 粗边框，禁止引入标准蓝/绿/红等杂色。
2. **收敛操作，卡片即实体**：整张卡片就是一个可点击的入口，不堆砌冗余按钮，依靠 Hover 状态暗示交互。
3. **信息降噪**：扫一眼必须只看到“学什么（标题）”、“依据什么（源码标签）”、“学到哪了（进度）”。

------

## 一、 信息结构与排版规范 (Data Hierarchy)

完全摒弃现有的扁平文本堆砌，重构为四级视觉层级：

- **第一级（最核心，纯白发光）**：模块标题（如：*我不知道该怎么做*）。使用 `text-white font-medium text-lg`。若模块已通关，标题右侧带一个极小的 `✨` 或青色 `✓`。
- **第二级（数据源胶囊标签）**：源码名称。废弃纯文本，改为精致的微小胶囊标签（Pill）。样式：`bg-slate-800/60 text-slate-300 text-xs px-2 py-1 rounded-md border border-slate-700/50`。
- **第三级（相对时间）**：最后学习时间。放在左下角，使用极小灰色字体 `text-slate-500 text-xs`（如：*2 hours ago*, *Just now*）。
- **第四级（进度数字）**：右上角显示精简进度，如 `[ 3/5 ]`，颜色使用 `text-cyan-600`。

------

## 二、 全息视觉与状态规范 (Aesthetic & States)

### 1. 卡片基础质感（全息玻璃）

- **背景**：`bg-slate-900/40 backdrop-blur-md`（透出底层的星空）。
- **边框**：极细的半透边框 `border border-slate-800/60`，圆角 `rounded-xl`。

### 2. 状态映射（通过进度底边展示，而非粗边框）

放弃那个 AI 提议的粗边框和卡片内画折线图，采用最优雅的**“底边进度条 (Bottom Edge Progress)”**：

- 在卡片的绝对底部（Bottom: 0），添加一条高度仅 `2px` 的线条。
- **进行中**：底边线条宽度根据进度计算（如 60%），颜色为发光的 `bg-cyan-500 shadow-[0_0_8px_#06b6d4]`。
- **已完成**：底边线条 100% 满格，颜色转为极亮的 `bg-cyan-400`。
- **未开始**：无底边线条，整体呈现未激活的深灰色调。

### 3. Hover 悬浮交互（物理隐喻）

- **浮力与发光**：鼠标悬浮时，卡片整体平滑上浮 `-translate-y-1`。边框瞬间被青色点亮 `border-cyan-500/40`，并带有微弱外发光 `shadow-[0_8px_30px_rgba(34,211,238,0.1)]`。
- **CTA 按钮浮现**：卡片右下角默认隐藏，Hover 时平滑浮现一个亮青色的文字链 `[ Open Module ↗ ]`，暗示点击整个卡片即可进入。

------

## 三、 高阶功能闭环 (UX & Management)

*完全吸纳另一个 AI 的优秀交互建议，并匹配极简风格：*

### 1. 右键上下文菜单 (Context Menu)

- 右键卡片弹出极简暗黑菜单（玻璃拟物态）。
- **选项**：
  1. `Rename`：就地变成输入框修改标题。
  2. `Duplicate & Ask New`：基于该源码新建一个提问模块。
  3. `Delete`：高危操作，悬浮时文字变暗红，点击后需弹窗二次确认。

### 2. 宏观调度台（排序与搜索）

- 在 Library 标题右侧或下方，提供极其低调的检索栏。
- **搜索**：无边框输入框，底层加 `border-b border-slate-700`，支持按模块标题或源码名称实时过滤。
- **排序 (Sort by)**：极简下拉菜单，支持 `Recently opened`（默认）、`Created date`、`Progress`。

### 3. 空状态闭环 (Empty State)

- 若无任何模块，屏幕中央显示幽灵态的星轨插图（可选）。
- 文字提示：`No learning modules yet. The galaxy is empty.`
- 主操作按钮：`[ Attach a Source to Begin ]`，点击直接跳转至 Start 页。

------

## 四、 绝对禁止项 (Red Lines)

1. **禁止开发白天模式**：当前 Sprint 绝对锁定暗黑星空模式，不消耗精力做亮色适配。
2. **禁止多重按钮并排**：卡片内绝对禁止出现两个以上的显性按钮互相抢夺注意力。
3. **禁止超过 1px 的边框**：所有边框必须是 `border` 或 `border-b`，严禁使用 `border-2` 破坏清冷质感。
4. **禁止破坏现有网格**：维持原有的 CSS Grid 布局，仅改变内部渲染。
