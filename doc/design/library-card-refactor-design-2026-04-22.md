# Echowhy Learning Library 卡片重构设计

## 背景

当前 `Learning Library` 页面已经具备模块展示、重命名和打开能力，但实现仍集中在单个页面文件中，卡片信息层级与状态表达较弱，管理能力也比较分散。

本次重构聚焦于学习库卡片，不改变已有模块存储结构，不改变整体网格布局，只对页面结构、卡片信息层级和交互闭环做增强。

## 设计目标

- 让用户一眼看到：学什么、基于什么、学到哪了
- 保留 `dark/light` 双主题，不牺牲现有主题系统
- 保留整卡作为主入口的简单交互
- 为 `Ask New` 提供清晰且独立的次动作
- 保持页面视觉风格与当前星空学习体验一致

## 取舍

### 保留

- 整卡可点击，默认执行 `Open`
- 源码标签 pill 化
- 右上角数字进度
- 卡片底边进度条
- 相对时间
- 搜索、排序、重命名、删除
- 双主题适配

### 不采纳

- AI 自动摘要
- 环形进度条
- 同时保留整卡点击与显性 `Open` 按钮
- 为了追求“极简暗黑”而移除白天主题

## 结构方案

采用 feature-first 方式拆分 Library：

- `features/learning-library/`
- 页面仅负责组装数据与顶层状态
- 卡片列表、卡片项、菜单、删除确认、排序搜索工具收敛到 feature 内

## 卡片信息层级

1. 标题
2. 源码 pill 标签
3. 相对时间
4. 右上角进度数字
5. 底边进度条
6. 次动作 `Ask New`

## 交互规则

- 点击卡片空白区域：打开模块
- 点击 `Ask New`：跳回 Start，并带上当前模块对应 source 作为新的提问起点
- 菜单支持：重命名、删除
- 删除需要二次确认

## 实施顺序

1. 抽离数据映射与工具函数
2. 重构卡片组件与列表组件
3. 接入搜索、排序、菜单与删除确认
4. 验证导航、主题、构建结果
